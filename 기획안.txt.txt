[프로젝트 개요: 동안비법 (AI 헤어 진단 및 솔루션 웹앱 MVP)]
본 프로젝트는 여성을 위한 AI 기반 헤어 진단 및 솔루션 서비스로, 사용자의 얼굴형과 평소 헤어 고민을 분석하여 맞춤형 헤어스타일을 제안하고, 최종적으로 자사(ODM) 뷰티 제품 구매로 연결하는 종합 뷰티 커머스 웹앱입니다. 직관적이고 세련된 UI/UX를 적용하여 전 연령대의 사용자가 쉽게 접근하고 몰입할 수 있도록 구성합니다.

[핵심 유저 플로우 (Flow)]

메인 랜딩: 서비스 소개 및 '분석 시작하기' 버튼 노출.

사전 설문 (2~4페이지): 사용자의 평소 헤어 고민거리와 문제점을 체크리스트 형태로 수집.

사진 업로드 및 가이드: 정면 얼굴 사진 업로드. (원본 사진은 보존하고 헤어스타일만 AI로 변환할 예정).

스타일 선택: 가슴선, 쇄골선, 어깨선, 단발, 턱선위, 숏 등 원하는 기장 1가지 선택. (다른 기장을 보려면 다시 시작).

AI 로딩 및 리워드: 이미지가 합성되는 동안(약 15~18초) 리워드/전면 광고를 노출하여 서버비 방어 및 수익 창출.

진단 결과 및 커머스 연동: AI 변환 이미지, 사전 설문 기반의 태그/진단 리포트 노출. 하단에 진단 결과와 매칭되는 자사 브랜드 제품(뿌리 볼륨 에센스, 새치 마스카라 등)을 노출하여 판매 유도.

[UI/UX 및 디자인 가이드 (첨부된 레퍼런스 폴더 참고)]

전체적인 톤앤매너는 너무 밝은 차홍 스타일만 고집하지 않고, 세련된 다크 톤(Dark Mode)이나 대비가 확실한 모던 UI를 혼합하여 적용.

진행 상태바: '1. 사진 추가 - 2. AI 합성 - 3. 결과 확인' 같은 직관적인 스텝 인디케이터를 화면에 배치.

해시태그 UI: 진단 결과나 선택 항목을 보여줄 때 #see_through, #natural_black 같은 둥근 배지(Badge) 형태의 UI를 적극 활용.

[7대 코딩 핵심 지시사항]
1. 비용 0원, 클라이언트 사이드 얼굴 분석: 비싼 백엔드 API 대신 브라우저단에서 MediaPipe(Face Mesh)나 face-api.js를 사용해 사용자의 스마트폰 리소스로 얼굴형을 분석.
2. 카메라 UX (오버레이 및 크롭 기능):

화면 중앙에 사람 얼굴 형태의 점선 가이드라인(Face Outline) 배치.

사용자가 사진을 올리면 핀치(Pinch)하여 크기를 조절하고 드래그하여 위치를 맞출 수 있는 크롭(Crop) UI 제공.

"정면·이마/귀 노출·밝은 조명" 등의 팁(Tip) 안내 문구 삽입.
3. 극한의 이미지 압축: 외부 합성 API 전송을 대비해, 업로드된 사진은 프론트엔드에서 가로세로 512px 수준으로 리사이징 및 용량 압축.
4. 철벽 캐싱(Caching) 시스템: '뒤로 가기'나 '새로고침' 시 중복 API 호출 방지를 위해 Local Storage 등에 상태 임시 저장.
5. CSV 기반 커머스 매칭 로직: 폴더 내 CSV(설문-결과-제품 매핑)를 파싱하여, '설문 답변'과 '얼굴형'을 대조한 뒤 정확한 자사 제품(볼륨 에센스 등)을 노출.
6. 확장성 (Premium): MVP 이후 측면/후면 사진 기반의 Vision API 분석 모듈이 추가될 수 있도록 컴포넌트 구조를 유연하게 설계.
7. 바이럴 메타데이터: Next.js Metadata API를 통한 Open Graph(OG 태그) 설정 및 진단 결과 페이지 하단에 카카오톡 SDK 기반 '결과 공유하기' 기능 구현.
8. 동적 프롬프트(Dynamic Prompt) 및 세밀한 레퍼런스 매칭 로직:
단순한 기장 변환을 넘어, 연령대와 개인의 헤어 고민이 최종 AI 합성 이미지에 반드시 반영되어야 해.

사용자의 사전 설문 데이터(예: 60대, 정수리 볼륨 꺼짐, 가는 모발 등)를 변수로 받아, AI 합성 API(Stable Diffusion 등)로 전송할 텍스트 프롬프트에 자동으로 조합/주입하는 로직을 구현해 줘. (예: [age] years old woman, [hair_problem] solution...)

단순히 '단발' 사진을 레퍼런스로 넘기는 것이 아니라, CSV 매핑 테이블을 참조하여 사용자의 '연령대'와 '고민(볼륨 등)' 조건에 정확히 부합하는 특정 레퍼런스 이미지(예: 60대 볼륨 단발 사진)가 선택되도록 알고리즘을 고도화해 줘.


[동안비법 프로젝트 MVP 상세 기획안]

1. 프로젝트 개요
- 본 프로젝트는 여성을 위한 AI 기반 헤어 진단 및 솔루션 서비스(웹앱 MVP)입니다.
- 사용자의 얼굴형과 평소 헤어 고민을 분석하여 맞춤형 헤어스타일을 제안하고, 최종적으로 자사(ODM) 뷰티 제품 구매로 연결하는 종합 뷰티 커머스입니다.
- UI/UX 톤앤매너: 청담동 하이엔드 뷰티 살롱 느낌. 세련된 다크 모드(Deep Charcoal) 또는 대비가 확실한 모던 UI 혼합 적용. 전 연령대(마케팅 코어: 40~70대) 가독성을 위해 큰 글씨, 넉넉한 여백, 64px 높이 버튼 적용.

2. 핵심 유저 플로우 (Flow)
- 메인 랜딩 -> 사전 설문(5페이지 15문항 정밀 문진 시스템 V3) -> 사진 업로드 및 가이드 -> 스타일 선택(기장 1가지) -> AI 로딩 및 리워드(15~18초) -> 진단 결과 및 커머스 연동

3. 우선순위 알고리즘 (surveyData.ts 연동)
- 복수 선택 문항에 가중치를 두어 Array Sorting을 거친 후 마스터 데이터와 대조.
- Q2 헤어 고민: 정수리 볼륨(1) → 넓은 이마·M자(2) → 새치·뿌리(3) → 옆두상 볼륨(4) → 엉킴·손상(5)
- Q15 볼륨 위치: 정수리(1) → 전체(2) → 앞머리(3) → 뒤통수(4)
- prioritize()가 선택 즉시 가중치순으로 정렬해 state에 저장하고, 실시간 순위를 배지(①②③)로 표시. 완료 화면(결과지)에서도 이 우선순위 요약 카드를 보여주어야 함.

4. 결과지 페이지 (Results Page) 개발 핵심 요구사항
- AI 변환 이미지 노출 영역 (현재는 준비된 SVG 라인아트 또는 Illustrations.tsx 활용, 추후 next/image 교체 가능하도록 설계)
- 사전 설문 기반의 태그/진단 리포트 노출 (해시태그 UI: #see_through, #natural_black 등 둥근 배지 형태 적극 활용)
- 동적 프롬프트 생성 로직: 사용자의 설문 데이터(연령, 헤어 고민, 가는 모발 등)를 변수로 받아 Stable Diffusion 등 AI API로 전송할 텍스트 프롬프트를 자동으로 조합/생성하는 함수 구현 (예: `[age] years old woman, [hair_problem] solution, voluminous roots...`)
- 레퍼런스 매칭 알고리즘: 단순히 '단발' 기장만 매칭하는 것이 아니라, CSV/마스터 테이블 구조를 참조하여 사용자의 '연령대'와 '핵심 고민(볼륨 등)' 조건에 정확히 부합하는 레퍼런스 스타일이 매칭되도록 로직 구현 (예: 60대 + 정수리 꺼짐 -> '60대 볼륨 단발 스타일' 매칭)
- 커머스 연동 (Soft-selling): 진단 결과(얼굴형 + 핵심 고민 우선순위)와 매칭되는 자사 브랜드 제품(뿌리 볼륨 에센스, 새치 마스카라 등)을 전문가의 처방전처럼 자연스럽게 노출하여 판매 유도.
- 바이럴 및 공유: Next.js Metadata API 기반 Open Graph(OG) 설정 가이드라인 및 카카오톡 SDK 기반 '결과 공유하기' 버튼 UI 배치.
- 확장성: MVP 이후 측면/후면 사진 분석 Vision API 모듈이 추가될 수 있도록 구조적인 유연성 확보.
