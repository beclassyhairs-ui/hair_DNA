# 어뷰티(A-Beauty) — 사업주 직접 설정 안내서

> 이 문서는 **개발 지식이 없어도** 그대로 따라 할 수 있게 만든 안내서입니다.
> 개발자가 코드로 대신 해줄 수 없는, **사업주 본인 계정으로만 할 수 있는 설정 3가지**를 다룹니다.
>
> - 대상 서비스 주소(프로덕션): **https://hair-dna.vercel.app**
> - 총 소요 시간: 약 **35~50분** (처음이면 여유 있게 1시간)
> - 준비물: 카카오 계정, Vercel 계정, Supabase 계정, 메모장(값 임시 보관용)

---

## 📋 전체 체크리스트 (순서대로 진행)

순서가 중요합니다. **1번에서 얻은 값이 2번에 들어가고**, 3번은 그와 별개지만 로그인이 실제로 동작하려면 **셋 다** 끝나야 합니다.

**1단계 — 카카오 개발자 콘솔** (약 15~20분)
- ☐ 1-1. developers.kakao.com 로그인 → 내 애플리케이션 → 앱 선택(또는 앱 만들기)
- ☐ 1-2. 앱 이름 확인 (동의창에 그대로 노출됩니다)
- ☐ 1-3. [카카오 로그인] 활성화 ON
- ☐ 1-4. [간편로그인] ON
- ☐ 1-5. Redirect URI 2개 등록 (프로덕션 + 로컬)
- ☐ 1-6. Client Secret 발급 + "사용함"으로 설정
- ☐ 1-7. **REST API 키** 복사 → 메모장에 `REST` 라고 적어두기
- ☐ 1-8. **JavaScript 키** 복사 → 메모장에 `JS` 라고 적어두기
- ☐ 1-9. 동의항목 최소화 (닉네임만 선택동의, 나머지 끄기)

**2단계 — Vercel 환경변수** (약 10분) — *1단계 완료 후에 진행*
- ☐ 2-1. USER_SESSION_SECRET 용 랜덤 문자열 새로 만들기 (ADMIN_SECRET과 다른 값)
- ☐ 2-2. Vercel → 프로젝트 → Settings → Environment Variables 진입
- ☐ 2-3. 변수 5개 등록 (KAKAO_REST_API_KEY / KAKAO_CLIENT_SECRET / KAKAO_REDIRECT_URI / USER_SESSION_SECRET / NEXT_PUBLIC_KAKAO_APP_KEY)
- ☐ 2-4. ⚠️ **Redeploy(재배포)** 실행 — 이걸 안 하면 아무것도 반영되지 않습니다

**3단계 — Supabase SQL 실행** (약 10분) — *1·2단계와 독립적이지만 반드시 아래 순서로*
- ☐ 3-1. 백업(스냅샷) 확인
- ☐ 3-2. `supabase/users_auth_schema.sql` 실행 — **반드시 먼저**
- ☐ 3-3. `supabase/hair_usage_schema.sql` 실행 — **위 파일 다음에**
- ☐ 3-4. Table Editor에서 테이블 4개 생겼는지 확인

**마무리**
- ☐ 4-1. 실제로 로그인 한 번 해보기 (아래 "다 끝나고 확인하는 법")

# 1단계. 카카오 개발자 콘솔 설정

> **소요 시간**: 약 15~20분
> **준비물**: 카카오 계정(사업주 본인 계정 권장), 값을 임시로 적어둘 메모장

### 이 단계가 하는 일 (한 줄)
손님이 "카카오로 시작하기"를 눌렀을 때, 카카오가 우리 사이트를 **신뢰할 수 있는 곳으로 인식**하게 등록하는 작업입니다.

---

## 1-1. 콘솔 접속하고 앱 찾기

1. 브라우저에서 **https://developers.kakao.com** 접속
2. 오른쪽 위 **[로그인]** 클릭 → 카카오 계정으로 로그인
3. 상단 메뉴에서 **[내 애플리케이션]** 클릭
4. 화면에 앱 목록이 나옵니다.
   - **앱이 이미 있으면** → 그 앱 카드를 클릭해서 들어갑니다.
   - **앱이 하나도 없으면** → **[애플리케이션 추가하기]** 버튼 클릭 →
     - `앱 이름`: 손님에게 보여줄 서비스 이름 (예: `어뷰티`)
     - `사업자명`: 사업자명 또는 본인 이름
     - `카테고리`: 뷰티/미용 계열 선택
     - 약관 동의 체크 → **[저장]**

> 💡 **앱 이름은 손님 화면에 그대로 나옵니다.**
> 손님이 로그인할 때 뜨는 동의 창에 "**〇〇〇에 로그인합니다**" 형태로 앱 이름이 노출됩니다.
> 임시 이름(테스트, test 등)으로 두면 손님이 그대로 보게 되니, **실제 서비스명으로** 지어주세요.
> 나중에 [앱 설정] → [앱] 화면에서 언제든 바꿀 수 있습니다.

---

## 1-2. 카카오 로그인 켜기

1. 앱에 들어간 뒤, **왼쪽 사이드 메뉴**에서 **[카카오 로그인]** 클릭
2. 맨 위에 **[활성화 설정]** 이라는 항목이 있습니다. 그 옆 스위치를 **ON**으로 바꿉니다.
   - 확인 팝업이 뜨면 **[확인]**
3. 같은 화면 조금 아래 **[간편로그인]** 항목의 스위치도 **ON**으로 바꿉니다.
   - *간편로그인(카카오톡 앱으로 바로 로그인되는 기능)이 켜져 있어야 휴대폰에서 로그인이 매끄럽습니다.*

---

## 1-3. Redirect URI 등록 (가장 중요 — 오타 주의)

**Redirect URI**(리다이렉트 URI)란: 손님이 카카오에서 로그인을 마치면, 카카오가 손님을 **다시 우리 사이트 어디로 돌려보낼지** 알려주는 주소입니다. 여기에 등록된 주소가 아니면 카카오가 **아예 거부**합니다.

1. 같은 **[카카오 로그인]** 화면을 아래로 스크롤하면 **[Redirect URI]** 항목이 있습니다.
2. **[Redirect URI 등록]** 버튼 클릭
3. 입력창에 아래 **두 개**를 각각 등록합니다. (한 줄에 하나씩, 또는 [+ 추가] 눌러 두 칸으로)

**① 프로덕션용 (실제 손님이 쓰는 주소) — 필수**
```
https://hair-dna.vercel.app/api/auth/kakao/callback
```

**② 로컬 개발용 (개발자가 테스트할 때 사용) — 함께 등록 권장**
```
http://localhost:3000/api/auth/kakao/callback
```

4. **[저장]** 클릭

> ⚠️ **복사·붙여넣기로만 입력하세요.** 손으로 타이핑하면 오타가 납니다.
> - 맨 앞이 `https://` 인지 (①번은 s가 붙고, ②번 localhost는 `http://` 로 s가 없습니다)
> - 맨 뒤에 `/` 를 **추가로 붙이지 마세요**
> - 앞뒤에 공백이 딸려 들어가지 않았는지
>
> 이 주소는 나중에 2단계의 `KAKAO_REDIRECT_URI` 값과 **글자 하나까지 똑같아야** 합니다.
> 다르면 로그인 시 `KOE006` 같은 오류가 뜹니다.

---

## 1-4. Client Secret 발급하고 "사용함" 켜기

**Client Secret**(클라이언트 시크릿)이란: 우리 서버만 아는 **추가 비밀번호**입니다. 키가 유출돼도 이게 없으면 남이 우리 서비스인 척할 수 없어서, 켜두는 편이 안전합니다.

1. 왼쪽 사이드 메뉴 **[카카오 로그인]** 하위에 있는 **[보안]** 탭 클릭
   *(메뉴 위치: 카카오 로그인 → 보안)*
2. **[Client Secret]** 항목의 **[코드 생성]** 버튼 클릭 → 긴 문자열이 하나 생성됩니다.
3. 생성된 문자열을 **복사해서 메모장에 `SECRET` 이라고 적어두세요.**
   - ⚠️ 이 값은 **지금 한 번만 편하게 볼 수 있습니다.** 꼭 복사해두세요.
4. 바로 아래 **[활성화 상태]** 를 **"사용함"** 으로 바꾸고 **[저장]**
   - "사용안함"으로 두면 값을 만들어도 적용되지 않습니다.

---

## 1-5. 키 2개 복사하기 ⚠️ 가장 헷갈리는 지점

카카오는 키를 **여러 종류** 주는데, 우리는 그중 **2개**를 씁니다. **서로 용도가 완전히 다릅니다.**

1. 왼쪽 사이드 메뉴 **[앱 설정]** → **[앱 키]** 클릭
2. 화면에 4종류가 표시됩니다: `네이티브 앱 키`, `REST API 키`, `JavaScript 키`, `Admin 키`
3. 이 중 **2개만** 복사합니다.

| 콘솔 화면에 적힌 이름 | 메모장 표기 | 나중에 들어갈 환경변수 | 용도 | 성격 |
|---|---|---|---|---|
| **REST API 키** | `REST` | `KAKAO_REST_API_KEY` | 카카오 **로그인** 처리 | 🔒 서버 전용 (외부 노출 금지) |
| **JavaScript 키** | `JS` | `NEXT_PUBLIC_KAKAO_APP_KEY` | 결과지 **카카오톡 공유** 버튼 | 🌐 브라우저에 노출돼도 되는 키 |

> ⚠️ **이 둘을 바꿔 넣는 실수가 가장 흔합니다.**
> - `KAKAO_REST_API_KEY` 자리에 JavaScript 키를 넣으면 → **로그인이 안 됩니다**
> - `NEXT_PUBLIC_KAKAO_APP_KEY` 자리에 REST API 키를 넣으면 → **카톡 공유가 안 되고, 서버 전용 키가 외부에 노출됩니다** (더 위험)
>
> 외우기: **"REST = 로그인(뒤에서), JavaScript = 공유(앞에서)"**

> ❌ **Admin 키와 네이티브 앱 키는 이 프로젝트에서 쓰지 않습니다.** 복사하지 마세요.
> 특히 Admin 키는 최고 권한 키라 절대 어디에도 넣으면 안 됩니다.

---

## 1-6. 동의항목 최소화

**동의항목**이란: 손님이 로그인할 때 "이 정보를 제공하는 데 동의하시겠습니까?"라고 묻는 목록입니다.

1. 왼쪽 사이드 메뉴 **[카카오 로그인]** → **[동의항목]** 클릭
2. 아래 기준으로만 설정합니다.

| 항목 | 설정 | 이유 |
|---|---|---|
| **닉네임** (프로필 정보) | **선택 동의** | 인사말에 이름을 쓰기 위함. 손님이 거절해도 로그인은 정상 동작합니다. |
| **프로필 사진** | 선택 동의 (또는 끔) | 없어도 무방 |
| **카카오계정(이메일)** | **끔** | 우리 서비스는 이메일을 쓰지 않습니다 |
| 전화번호 / 성별 / 연령대 / 생일 / 배송지 등 | **전부 끔** | 수집하지 않습니다 |

> 💡 **회원번호(고유 ID)는 목록에 없어도 자동으로 받습니다.** 별도 설정이 필요 없습니다.
> 우리 서비스는 이 회원번호 하나만으로 손님을 구분합니다.

> ⚠️ **필요 없는 항목을 켜지 마세요.**
> - 손님 입장에서 동의 창에 요구 항목이 많으면 **로그인을 포기**합니다. (이탈률 직결)
> - 수집한 개인정보는 법적으로 **보관·파기 책임**이 따라옵니다. 안 받는 게 가장 안전합니다.
> - 이메일·전화번호 등 일부 항목은 카카오의 **별도 검수**를 받아야 해서 승인 지연도 생깁니다.

```
SECRET : (Client Secret 값)
```

> 카카오톡·이메일·메신저로 이 값을 보내지 마세요.

# 2단계. Vercel 환경변수 등록

> **소요 시간**: 약 10분
> **준비물**: 1단계에서 적어둔 메모장(REST / JS / SECRET), Vercel 계정
> **선행 조건**: 1단계를 먼저 끝내야 합니다.

### 이 단계가 하는 일 (한 줄)
1단계에서 받은 키들을 **우리 사이트 서버에 안전하게 저장**해서, 서버가 카카오와 대화할 수 있게 만듭니다.

**환경변수**란: 코드 안에 직접 적으면 위험한 비밀 값(비밀번호·키 등)을 **서비스 설정 화면에 따로 보관**하는 방식입니다.

---

## 2-0. 먼저 USER_SESSION_SECRET 만들기

이 값 하나만은 **어디서 받아오는 게 아니라, 직접 만들어야 합니다.**

**이게 뭔가요?** 손님이 로그인한 뒤 "이 사람은 로그인한 사람이 맞다"는 것을 우리 서버가 증명하는 데 쓰는 **도장(서명키)** 입니다. 이 값이 유출되면 남이 다른 손님인 척할 수 있습니다.

**만드는 법 — 아래 중 편한 방법 하나**

- **방법 A (권장·가장 쉬움)**: 무작위 문자열 생성 사이트 이용
  1. 검색창에 `random password generator` 또는 `랜덤 문자열 생성기` 검색
  2. 길이를 **40자 이상**(가능하면 64자)으로 설정
  3. 영문 대소문자 + 숫자 포함으로 생성 → 복사

- **방법 B**: 아무 키나 마구 눌러 만들기
  - 영문·숫자를 섞어 **최소 40자 이상**. 의미 있는 단어(가게 이름, 생년월일, 전화번호)를 넣지 마세요.

- **방법 C (개발자용)**: 터미널에서 `openssl rand -base64 32`

> 🔴 **반드시 지켜야 할 규칙: `ADMIN_SECRET`과 절대 같은 값을 쓰지 마세요.**
> `ADMIN_SECRET`은 사업주용 관리자 화면 비밀번호이고, `USER_SESSION_SECRET`은 손님 로그인용 도장입니다.
> 두 값이 같으면 **손님 세션과 관리자 세션이 섞일 위험**이 생깁니다.
> 두 값은 반드시 **완전히 다른, 서로 무관한 문자열**이어야 합니다.

> 💡 한 번 정하면 **바꾸지 마세요.** 나중에 바꾸면 이미 로그인해 있던 손님이 **전부 로그아웃**됩니다.

---

## 2-1. Vercel 환경변수 화면 찾아가기

1. **https://vercel.com** 접속 → 로그인
2. 로그인하면 프로젝트 목록이 보입니다. **어뷰티 프로젝트**(hair-dna)를 클릭
3. 프로젝트 화면 위쪽 탭 중 **[Settings]** 클릭
4. 왼쪽 사이드 메뉴에서 **[Environment Variables]** 클릭
5. 변수를 추가하는 입력 폼이 나타납니다:
   - **Key** (또는 Name) — 변수 이름 넣는 칸
   - **Value** — 실제 값 넣는 칸
   - **Environments** — 어디에 적용할지 고르는 체크박스 (Production / Preview / Development)

---

## 2-2. 변수 5개 등록

아래 표를 보고 **한 개씩 5번** 반복합니다.
매번: `Key` 입력 → `Value` 붙여넣기 → `Environments` 체크 → **[Save]** 클릭

| # | Key (그대로 복사) | Value에 넣을 것 | Environments |
|---|---|---|---|
| 1 | `KAKAO_REST_API_KEY` | 메모장의 **REST** 값 | Production ✔ Preview ✔ Development ✔ |
| 2 | `KAKAO_CLIENT_SECRET` | 메모장의 **SECRET** 값 | Production ✔ Preview ✔ Development ✔ |
| 3 | `KAKAO_REDIRECT_URI` | `https://hair-dna.vercel.app/api/auth/kakao/callback` | **Production만 ✔** |
| 4 | `USER_SESSION_SECRET` | 2-0에서 직접 만든 랜덤 문자열 | Production ✔ Preview ✔ Development ✔ |
| 5 | `NEXT_PUBLIC_KAKAO_APP_KEY` | 메모장의 **JS** 값 | Production ✔ Preview ✔ Development ✔ |

### 각 변수 상세

**1. `KAKAO_REST_API_KEY`**
- 어디서 얻나: 카카오 콘솔 → 앱 설정 → 앱 키 → **REST API 키**
- 형식 예시: `a1b2c3d4e5f6...` (영문소문자+숫자 32자 정도)
- 하는 일: 카카오에게 "우리가 그 앱이다"라고 밝히는 신분증
- ⚠️ JavaScript 키와 헷갈리지 마세요

**2. `KAKAO_CLIENT_SECRET`**
- 어디서 얻나: 카카오 콘솔 → 카카오 로그인 → **보안** → Client Secret
- 형식 예시: `Xy7Qw...` (영문 대소문자+숫자 32자 정도)
- 하는 일: 위 신분증에 딸린 추가 비밀번호
- 콘솔에서 **"사용함"** 으로 설정돼 있어야 합니다 (1-4 참고)

**3. `KAKAO_REDIRECT_URI`**
- 어디서 얻나: 직접 입력 — 아래 값을 **그대로** 복사
  ```
  https://hair-dna.vercel.app/api/auth/kakao/callback
  ```
- 하는 일: 로그인 후 손님이 돌아올 주소. **1-3에서 카카오 콘솔에 등록한 값과 글자 하나까지 같아야** 합니다.
- 왜 Production만? 이 주소는 실제 서비스 주소입니다. Preview/Development에는 넣지 않으면 시스템이 그 환경의 주소로 알아서 조립합니다. (헷갈리면 Production에만 체크하세요.)
- ⚠️ 맨 뒤에 `/`를 붙이지 마세요. 앞뒤 공백도 없어야 합니다.

**4. `USER_SESSION_SECRET`**
- 어디서 얻나: **직접 만듦** (2-0 참고)
- 형식 예시: 40자 이상 영문+숫자 무작위 문자열
- 하는 일: 손님 로그인 상태를 증명하는 도장
- 🔴 `ADMIN_SECRET`과 **반드시 다른 값**

**5. `NEXT_PUBLIC_KAKAO_APP_KEY`**
- 어디서 얻나: 카카오 콘솔 → 앱 설정 → 앱 키 → **JavaScript 키**
- 형식 예시: `9f8e7d6c...` (영문소문자+숫자 32자 정도)
- 하는 일: 손님이 진단 **결과지를 카카오톡으로 공유**할 때 사용
- 참고: `NEXT_PUBLIC_`으로 시작하는 변수는 **브라우저에 노출되도록 설계된 것**입니다. JavaScript 키는 원래 공개돼도 되는 키라 정상입니다.
- 이 값이 없어도 사이트는 안 깨집니다 — 카톡 공유 대신 링크 복사로 대체 동작합니다.

# 🔧 문제가 생겼을 때 — 증상별 원인표

당황하지 않으셔도 됩니다. 아래 대부분은 **값 하나 고치고 재배포**하면 해결됩니다.

| 증상 | 가장 흔한 원인 | 해결 방법 |
|---|---|---|
| 로그인 버튼을 눌러도 아무 일 없고, 화면에 **"카카오 로그인이 아직 설정되지 않았습니다"** | `KAKAO_REST_API_KEY`가 비어 있거나, 등록 후 **재배포를 안 함** | 2단계 표 확인 → **Redeploy 실행** (2-3) |
| 카카오 화면에서 **`KOE006`** 또는 "등록되지 않은 리다이렉트 URI" 오류 | 콘솔에 등록한 Redirect URI와 `KAKAO_REDIRECT_URI` 값이 **미세하게 다름** | 1-3과 2-2의 값을 나란히 놓고 비교. 맨 뒤 `/`, `http`/`https`, 공백, 오타 확인 |
| 카카오 화면에서 **`KOE101`** / "앱 키가 유효하지 않음" | REST API 키 자리에 **JavaScript 키**를 넣음 | 1-5 표를 다시 보고 두 키를 바로잡기 → 재배포 |
| 동의까지 했는데 **원래 화면으로 튕기고 주소에 `?login=failed`** 가 붙음 | ⓐ Supabase SQL 미실행(`users` 표 없음) ⓑ `USER_SESSION_SECRET` 미등록 ⓒ Client Secret 불일치 | ⓐ 3단계 실행 → ⓑ 2단계 4번 확인 → ⓒ 콘솔에서 Client Secret "사용함"인지, 값이 같은지 확인 |
| 로그인은 되는데 **합성 시 "오늘 무료 5회를 모두 사용했어요"가 처음부터** 뜸 | `hair_usage_schema.sql` 미실행 또는 실행 실패 | 3-3의 ② 실행 → Table Editor에 `hair_usage` 있는지 확인 |

| SQL 실행 시 **`already exists`** 문구 | 이미 만들어져 있음 | 대부분 정상입니다. Table Editor에서 표 4개가 보이면 그대로 두세요 |
| 로그인은 되는데 **잠시 뒤 자꾸 로그아웃**됨 | `USER_SESSION_SECRET` 값을 중간에 바꿈, 또는 환경별로 다른 값을 넣음 | 한 값으로 통일하고 재배포. 이후 손님들은 한 번 다시 로그인하면 정상 |
| 관리자 화면(`/admin`)이 갑자기 안 열림 | `USER_SESSION_SECRET`을 넣으면서 `ADMIN_SECRET`을 건드림 | `ADMIN_SECRET`은 원래 값 그대로여야 합니다. 두 값은 별개입니다 |
| Vercel 재배포가 **Error(빨간색)** 로 실패 | 코드 문제일 가능성 | 환경변수 탓이 아닐 수 있습니다. 실패 로그를 캡처해 개발자에게 전달하세요 (**값이 보이는 화면은 캡처 금지**) |

### 그래도 안 되면

개발자에게 전달할 때 아래 3가지만 알려주시면 빠릅니다.

1. **어느 단계**에서 막혔는지 (예: "2-3 재배포까지 했는데 확인 1에서 실패")
2. **화면에 뜬 오류 문구** 그대로 (예: `KOE006`)
3. **주소창의 URL** (예: `...?login=failed`)

> 🔒 **절대 함께 보내지 말아야 할 것**: 키 값, Client Secret, USER_SESSION_SECRET, 환경변수 화면 캡처.
> 이 값들은 없어도 원인 파악이 가능합니다.

## 참고 — 이 문서에 나오는 용어 정리

| 용어 | 쉬운 설명 |
|---|---|
| **OAuth / 카카오 로그인** | 우리가 비밀번호를 받지 않고, 카카오가 대신 본인 확인을 해주는 방식 |
| **Redirect URI** | 로그인을 마친 손님을 다시 데려올 우리 사이트 주소 |
| **REST API 키** | 우리 **서버**가 카카오와 대화할 때 쓰는 신분증 (외부 노출 금지) |
| **JavaScript 키** | 손님 **브라우저**에서 카톡 공유할 때 쓰는 키 (공개돼도 안전) |
| **Client Secret** | 신분증에 딸린 추가 비밀번호 |
| **환경변수** | 코드에 직접 못 적는 비밀 값을 따로 보관하는 설정 |
| **배포 / 재배포(Deploy/Redeploy)** | 바뀐 설정·코드를 실제 사이트에 반영하는 것 |
| **Supabase** | 손님 데이터를 저장하는 데이터베이스 서비스 |
| **SQL** | 데이터베이스에게 "이런 표를 만들어라"라고 지시하는 명령문 |
| **테이블(표)** | 엑셀 시트처럼 줄과 칸으로 데이터를 담는 단위 |
| **RLS** | 표에 거는 잠금장치. 우리 서버만 열 수 있게 막아둡니다 |
| **세션 / 쿠키** | 손님이 로그인 상태임을 브라우저가 기억하는 방식 (우리 서비스는 30일 유지) |
