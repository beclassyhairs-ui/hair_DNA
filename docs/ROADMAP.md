# A-Beauty ROADMAP — 작업 큐 (PM방 ↔ Claude Code 연락 노트)

> 이 파일은 PM방(Claude 채팅)에서 확정된 작업 순서·검토 결론을 Claude Code에 전달하는 큐다.
> 세션 시작 시 `docs/PROJECT_STATE.md`와 함께 읽을 것. **진척·구현 상태가 충돌하면 PROJECT_STATE.md가 우선**이고, 순서·정책이 충돌하면 이 파일이 우선.
> 갱신 주체: PM방(사업주 승인 후). Claude Code는 평소 이 파일을 직접 수정하지 않고, 완료 항목은 PROJECT_STATE.md에 기록한다.
> 마지막 갱신: 2026-07-20 — **사업주 지시로 P0~P3 우선순위 체계로 전면 재편**(기존 기능 작업 + 누락 숙제 18개 A~E 통합). 완료 표시는 PROJECT_STATE.md와 대조해 반영함.

## 0. 승인 관문

- 수정·커밋·일반 명령: 자동 승인 (settings.json). 파괴 명령(rm -rf, force push, reset --hard, clean -f)은 하드 차단.
- 사람 승인 3관문 유지: ① **push/배포 직전 확인** (push=Vercel 자동 배포이므로, CLAUDE.md 4번) ② **Supabase SQL은 사업주가 SQL Editor에서 직접 실행** (🟢조회/🔴변경 구분해 안내) ③ **상품 실노출(approved) 승인**.
- 백엔드/DB/인증/RLS/환경변수 변경은 커밋 전 Codex 검수 필수. '수정 필요' 판정이면 커밋 제안 금지.

## 5. 자동화 단계 정의 (사업주 합의)

- 1단계(완료): AI 팀 체계 — Claude Code+Codex 연동, CLAUDE.md 규칙.
- 2단계(진행 중): 소싱 파이프라인 수동 완성 — P0의 커머스 파이프라인 ~ E2E 게이트.
- 3단계: `sourcing_candidates` 테이블 + 인세인서치 자동 투입(하루 1회 10~20개, 그 이상 금지). 자동화 대상은 "운반"뿐 — keep/drop 판단·승인·노출은 전부 사람.
- 4단계: 오픈클로 정기 점검 — 등록 상품 URL 생존·이미지 깨짐 순찰. 문제 발견 시 자동으로 내리지 않고 "검토 필요" 플래그 + 사업주 보고.
- 5단계: 데이터 루프 — 주간 자동 리포트(클릭 톱/클릭 0/타입별 전환율) → 정리 후보 제시 → 소싱 키워드 역반영.
